Minimalist Stitch Counter

Helpful resource for any crochet or knitting project.

This app was created with Godot Engine in less than 24 hours. My goal going into the project was to create a single screen stitch counter so that the user never lost the ability to use the main counter function. Most stitch counters I've seen have distracting pop-ups or simply change views for various options. This app strikes a balance between functionality and aesthetics as it avoids these distractions. In addition to the single screen goal, I wanted the user to be able to change the color scheme of the app at any time.

A full user guide is available in the GitHub Repo. Users are able to save/load/delete up to 8 projects, choose between 9 color schemes, and cycle between a simple counter and a row counter that displays project progress based on stitches per row.

Queue by Queue

An increasingly difficult video game with unique memory and puzzling mechanics.

At first glance, this game is deceptively simple. Move the large blue squares to the small ones by utilizing the effects of the different colored squares. Sounds easy enough, but there is another aspect to the game that makes it fairly unique and occasionally frustrating.

In addition to understanding all the different mechanics of the tiles, the player must solve the entire puzzle without seeing anything move. All of the action inputs are placed in a queue and nothing happens on the board until the player 'executes' the queue. It can be surprisingly difficult to keep up with the state of the board, especially when certain tiles change the layout itself.

Most of the base mechanics are present and this is definitely a project I plan to revisit in order to improve the UI.

Elementary Cellular Automata

Pure generative output represented through simple rulesets.

The rules of cellular automata are easy enough to understand if you look at the output row by row. Each element's state is determined by the three elements directly above it in the preceding row. Changing around the ruleset for each permutation allows you to drastically affect the outcome.

Watercolor Clouds

Generate clouds with a watercolor style to them.

Each watercolor 'spot' starts with a randomly generated polygon as the base. After the initial shape is established, layers of translucent deviations are drawn over each other. The algorithmic variations in the layers result in different color intensities for each portion of the overall image.

This is such a great project because wildly different results can be achieved through small changes to the generation code.
