Crooked Rook

A custom chess engine completely completely from scratch in only one week, strangely inspired by an episode of Lost. I'm using P5.js for the graphics but the game logic is written solely by me in pure Javascript. No existing chess frameworks were used.

Take a look below the game to read more about how it was created, find out what I was able to accomplish in my one week timeframe, and see a video of me losing to the engine that I created.

Try to beat the Crooked Rook at his own game but be careful ... he cheats!

How It Works

The general idea of a Chess engine is for the AI to "look ahead" a certain number of moves in order to find the action that maximizes it's own position or strength. The complexity comes in when accounting for how the enemy, the human player, will try to minimize the AI's position.

When it becomes the AI's turn, it loops through each available move. If the desired depth of evaluation is greater than 0, then for every move the AI can make, it will also loop through each available move for the human player. If the depth permits, then the next stage is to evaluate each of the possible moves the AI could make in response.

The more moves ahead the AI can look, the stronger it will be. Unfortunately, there are just too many moves in the game of Chess for each possible scenario to be evaluated all the way to the end of the game. Luckily, a system of pruning is in place to limit the number of moves that are evaluated. The idea is to compare current positions against previously evaluated branches in order to determine if they're worth pursuing.

If you'd like to learn more about the algorithms that make all this possible, check out the source code for the Crooked Rook here or read about Alpha-Beta pruning here.

Accomplishments

I'm very proud of what I was able to accomplish in only one week. I started with essentially no knowledge of how Chess engines are designed and was able to create a system that can reliably beat me at Chess.

What I Could Have Improved

There are several decisions I made early on that prevented me from specific improvements later in the process. For example, I failed to add the simple capability of castling kings. Based on my structure for determining valid moves, it would have taken me at least two days of working time in order to add the ability.
